# Governance

SentryLoom is currently a maintainer-led open-source project.

## Decision making

- Routine fixes and documentation may be accepted through normal review.
- Security boundaries, remote protocols, destructive actions, privileges,
  cryptographic formats, and data collection require maintainer approval.
- Large changes should begin with a public design issue unless disclosure would
  reveal an unpatched vulnerability.
- Decisions prioritize user safety, local control, transparent behavior,
  interoperability, and conservative false-positive handling.

## Maintainers

Maintainers triage issues, review contributions, manage releases, coordinate
security response, and enforce community standards. Maintainer status is earned
through sustained, high-quality contributions and sound security judgment.

As the contributor base grows, governance may evolve toward multiple
maintainers and documented voting. Governance changes will be reviewed in
public.

## Project assets

The Apache 2.0 license covers source code and documentation. It does not grant
ownership of project infrastructure, signing keys, domains, social accounts,
or trademarks. Release-signing keys are never stored in the repository.
