PROTECTION POLICY
Local security controls

          
          
          
          
          
          
          
          
          
          
          
Connection monitoring observes endpoint metadata and DNS cache entries. It does not inspect packet payloads or TLS contents; that requires a signed Windows Filtering Platform or packet-capture driver.

        

        

          

            
DEVICE CONTROL
USB removable storage

            CHECKING
          

          
          
This uses the Windows “All Removable Storage classes: Deny all access” computer policy. Administrator approval is required. Reconnect storage devices after changing the policy; organization-managed Group Policy can override it.

        

        

          

            
CENTRAL MANAGEMENT
SentryLoom HQ

            STANDALONE
          

          
This endpoint is locally managed and does not send telemetry to a server.

          

            
            
          

          

            
            
            
            
            

              
              
            

            Enter or discover an HQ server, then submit it here.
          

          

            
            
            

              
              
            

            Primary method: ask the HQ administrator to generate a rotating maintenance password. Use this request only when immediate approval is available.
          

          
This device submits an enrollment request without a user credential. An HQ administrator must approve it before telemetry or remote actions begin. Managed mode sends security posture, protection health, scan metrics, update state, performance data, and event types. HQ can request only predefined SentryLoom actions—never arbitrary shell commands. Local protection continues when HQ is unavailable.

        

        

          

            
NETWORK PRIVACY
Ad-blocking DNS

            CHECKING
          

          
Choose a public filtering resolver. SentryLoom configures every currently active routed Windows adapter and enables encrypted DNS-over-HTTPS without plaintext fallback.

          

          

          

            
            
          

          
Windows will request administrator approval. Original per-adapter DNS settings are backed up before the first change and automatically restored if resolution verification fails. VPNs and browsers with their own Secure DNS can override Windows DNS.

Firewall enforcement uses the Microsoft-signed Windows Defender Firewall service and its WFP engine. SentryLoom does not install an unsigned kernel driver. Restart resident protection after changing collector settings.

Feeds are downloaded only from allowlisted HTTPS origins. Spamhaus DROP remains subject to its fair-use terms; CIRCL and Botvrij provide public MISP feeds; Linux Malware Detect signatures are GPLv2. VirusTotal and AndroZoo are not bundled because their free access is limited, and live malware repositories are never downloaded. Network blocking remains opt-in.

          

            Restricted and sample-based sources
            
VirusTotal has API quotas and public-API usage restrictions. AndroZoo is academic-only, capped, expiring, non-commercial, and non-redistributable. theZoo contains live malware rather than a maintained IOC feed. SentryLoom deliberately excludes these sources instead of putting malware samples or misleading “unlimited” claims on managed endpoints.
