# SentryLoom Endpoint Security v0.16.1 + SentryLoom HQ v0.4.1

## Windows antivirus, endpoint security, on-premises EDR, and fleet administration update

> **Unsigned preview:** These reproducible Windows installers are not yet
> Authenticode-signed. Windows SmartScreen may warn before execution. Verify
> the SHA-256 checksums below. Production and enterprise operators should build
> from source with an eligible code-signing certificate or wait for an

SentryLoom v0.16.1 is a security and reliability update for the open-source,
offline-first Windows antivirus and endpoint detection platform. It ships with
SentryLoom HQ v0.4.1, the optional self-hosted HTTPS fleet-management server.
This release repairs HQ setup-password authentication, makes local server
discovery substantially more reliable, creates explicit Windows Firewall
policies, adds verbose graphical installation progress, and introduces
rotating administrator-controlled maintenance authorization.

SentryLoom remains Apache-2.0 licensed for personal, research, educational,
internal business, managed-service, and commercial use.

## Release highlights

- Reliable administrator passwords selected during SentryLoom HQ installation.
- Automatic HQ discovery across every active Windows IPv4 network adapter.
- Explicit, verified, and removable Windows Firewall rules on clients and HQ.
- Detailed graphical progress for client and server installation.
- Visible and retryable endpoint enrollment requests.
- Rotating, expiring, use-limited, and immediately revocable maintenance
  passwords.
- RSA-encrypted endpoint maintenance requests with a strict 20-second approval
  window.
- Password protection for critical settings, protection disablement, HQ
  disconnection, and uninstall.

## SentryLoom HQ password repair

The HQ installer now transfers the chosen password through a temporary
installer-owned UTF-8 file rather than a command-line argument. PowerShell
loads the exact value, Node.js creates a fresh PBKDF2-SHA256 salt and hash, and
an independent verifier checks that the stored configuration accepts the
chosen password before HQ starts. Setup fails with a diagnostic rather than
claiming success if any stage disagrees.

Existing HQ installations can run the v0.4.1 installer as an upgrade. Their
database and TLS certificate remain intact, while the administrator password
is replaced by and verified against the value entered in Setup.

## Rotating endpoint maintenance authorization

HQ administrator-generated passwords are the primary authorization method:

- Cryptographically random 35-character values.
- Shown once and never stored as plaintext.
- SHA-256 hash storage in the HQ SQLite database.
- Administrator-selected 5, 10, 30, or 60-minute lifetime.
- Administrator-selected one, three, or ten-use policy.
- Immediate revocation from the HQ dashboard.
- Global fleet scope for administrator-generated passwords.
- Device scope and one-use policy for client-requested passwords.
- Rate limiting for invalid client authorization attempts.

Managed clients require a current maintenance password before:

- Changing endpoint protection, scanning, exclusion, or monitoring policy.
- Disabling realtime or behavioral protection.
- Changing encrypted DNS or removable-storage controls.
- Clearing SentryLoom firewall policy.
- Disconnecting the endpoint from HQ.
- Continuing through the supported Windows uninstaller.

For urgent interactive work, a client can request maintenance access. It
creates an ephemeral RSA-2048 key pair and asks HQ for approval. An
administrator has 20 seconds to approve or reject the request. HQ creates a
device-specific one-time password and encrypts it with RSA-OAEP-SHA256 before
delivery. The private key remains on the endpoint and is not stored by HQ.

## Network discovery and Windows Firewall

The endpoint now calculates the directed IPv4 broadcast address for every
active non-loopback adapter. It sends three discovery rounds to those
addresses, the global broadcast address, and loopback. This covers common
Ethernet, Wi-Fi, VPN, multi-homed, and mixed-subnet Windows environments more
reliably than a single `255.255.255.255` datagram.

Setup creates and verifies named firewall groups:

- `SentryLoom Endpoint`
  - Application-scoped TCP outbound access for HTTPS and threat intelligence.
  - Application-scoped UDP 32110 outbound access for LAN HQ discovery.
- `SentryLoom HQ`
  - TCP HTTPS inbound access from `LocalSubnet`.
  - UDP 32110 discovery inbound access from `LocalSubnet`.
  - UDP 32110 discovery responses outbound to `LocalSubnet`.

Rules support all Windows network profiles while retaining the `LocalSubnet`
boundary for HQ access. Both uninstallers remove their owned firewall rules.

## Installer experience and diagnostics

Both modern Inno Setup wizards now show timestamped operational detail while
keeping PowerShell, Node.js, winget, and scheduled-task windows hidden.

The client installer reports prerequisite detection, previous-version cleanup,
credential encryption, HQ discovery/enrollment, threat database updates,
scheduled protection registration, firewall configuration, and completion.

The HQ installer reports runtime detection, TLS/database initialization,
administrator-password hashing and verification, startup-task registration,
firewall validation, listener startup, and completion. Persistent diagnostics
are written to `%ProgramData%\SentryLoom HQ\Logs\install.log`.

## Enrollment interface repair

The endpoint Settings interface keeps **Request administrator approval**
available whenever the endpoint is not enrolled, including rejected and
pending retry states. Entering an HQ URL also reveals the action immediately.
Approved endpoints receive a separate maintenance panel for primary passwords
and optional 20-second approval requests.

### Endpoint backend

- `src/server.js` — enforcement for critical settings, DNS, USB, firewall,
