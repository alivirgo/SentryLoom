# SentryLoom v0.16.0 + HQ v0.4.0

## Open-source Windows antivirus, endpoint security, and on-premises EDR preview

> **Unsigned preview:** These first public Windows installers are reproducible
> development builds and are not Authenticode-signed. Windows SmartScreen may
> warn before execution. Verify the SHA-256 hashes below. Production and
> enterprise operators should build from source with their own trusted
> code-signing certificate or wait for an officially signed release.

See the project's [code signing policy](../../CODE_SIGNING_POLICY.md) and
[privacy disclosure](../../PRIVACY.md).

SentryLoom's first public release combines an offline-first Windows endpoint
security client with an optional self-hosted SentryLoom HQ management server.
The project is now available under Apache License 2.0 for personal, research,
education, internal business, hosted, and commercial use.

## Endpoint security highlights

- Local quick, full, custom, startup, process-image, and removable-drive malware
  scans.
- SHA-256 signatures, conservative heuristics, ClamAV verification, and
  community threat intelligence from MalwareBazaar, URLhaus, Feodo Tracker,
  and ThreatFox.
- AES-256-GCM encrypted quarantine and tamper-evident audit history.
- Realtime fixed-drive and Downloads monitoring.
- Process, persistence, ransomware-canary, Windows security-event, DNS, TCP,
  removable-media, and firewall-integrity monitoring.
- Local reputation lookup for hashes, domains, URLs, IPs, and network
  endpoints.
- Reversible DNS-over-HTTPS, USB storage, and high-confidence firewall controls.

## Windows desktop experience

- Native C# launcher with no Command Prompt or PowerShell window.
- Single-instance console restore when the executable is double-clicked.
- Notification-area icon: green while HQ is reachable and red otherwise.
- Bounded background command, scan, failure, and reconnect output in the
  Activity view.
- Responsive dependency-free dashboard with dark/light themes and actionable
  Windows notifications.

## SentryLoom HQ v0.4.0

- Self-hosted HTTPS fleet-management server with SQLite persistence.
- Certificate-pinned client enrollment and explicit administrator approval.
- Sanitized two-second endpoint telemetry and fleet posture dashboards.
- Offline, detection, collector-failure, and failed-command alerts.
- Allowlisted remote scans, protection repair, threat-database updates, and
  certificate-bound client updates—never an arbitrary remote shell.
- Installer-selected administrator password with PBKDF2-SHA256 storage.
- Resilient startup task and automatic client recovery after network loss,
  sleep, hibernate, VPN changes, server outages, and process restarts.

## Security and privacy

- Standalone endpoints require no cloud account and send no HQ telemetry.
- Runtime credentials are generated or entered during setup and encrypted or
  hashed locally.
- No API keys, private keys, certificates, customer data, databases, or
  generated credentials are included in the source repository.
- Signed update trust is bound to the exact package hash, version, certificate
  thumbprint, and certificate subject.
- GitHub secret scanning, push protection, CodeQL, Dependabot, and Windows CI
  are enabled for the public repository.
